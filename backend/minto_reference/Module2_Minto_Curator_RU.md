# Опорные тезисы из первоисточника (простым языком)

-   Одна главная мысль; начинаем с главного, а затем подкрепляем
    аргументами.

-   Связи в пирамиде: вверх/вниз/в стороны; верхний уровень резюмирует
    нижний; горизонтально --- идеи одного типа в логическом порядке.

-   Два вида логики: дедукция и индукция; типы последовательностей:
    хронологическая, структурная, сравнительная.

-   Введение (SCQR/«история»): контекст → затруднение → вопрос → ответ.

# Сценарий куратора (что говорить)

Пирамида = «главное → опоры → детали». На одном уровне --- идеи одного
типа и в выбранной последовательности.

Письма/CRM: Плохой: «сложности... водитель... возможно, перенесем».
Хороший: «Главное: доставим завтра 15:00--18:00... Просьба: подтвердите
окно».

Промпт к ИИ (шаблон): «Цель/роль ИИ → Контекст/исходные →
Формат/критерии ответа».

Экологично: вежливо, чётко, по фактам; критика приватно и с предложением
решения.

# Практика и материалы

**Практика 1 (CRM): переписать исходник.**

Исходник: «Привет. С доставкой неясно, машин не хватает. Клиент
нервничает. Надо бы что-то решить?»

Ожидание: 3--5 предложений по шаблону «Главное → Почему → Действия →
Просьба/срок».

**Практика 2 (аудио): сообщить руководителю о риске срыва срока (1--2
мин) по структуре «Главное → Причина → План → Просьба».**
